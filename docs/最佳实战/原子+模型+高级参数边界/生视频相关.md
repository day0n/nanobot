# AI Video Prompt Best Practices (Knowledge Base)

This document defines a reusable prompt framework for the following video generation models:

- Sora 2
- Veo 3.1 Fast
- Kling 3.0 Standard
- Kling 2.6 Pro
- Seedance 1.5 Pro
- Hailuo Video 2.3 Pro

---

## 1. Core Prompt Structure

```
[Camera & Lens]
+ [Motion / Physics]
+ [Lighting & Mood]
+ [Visual Effects]
+ [Style / Rendering]
+ [Shot Structure (optional)]
```

---

## 2. Design Principles

- Think like a director, not a describer  
- One shot = one main action + one camera + one lighting  
- Use physical motion instead of abstract verbs  
- Keep prompts modular and reusable  
- Shorter prompts = more creative freedom, longer prompts = more control

## 4. Shot Structure

### Multi-shot (Sora 2 only)

```
[00:00–00:03] Hook
[00:03–00:06] Build
[00:06–00:09] Peak
[00:09–00:12] Resolution
```

Rules:

- Separate shots clearly  
- Each shot = one action + one camera + one lighting  
- Avoid mixing multiple events in one shot  

---

## 5. Model-Specific Notes

### Sora 2
- Supports multi-shot and timeline prompts  
- Works well with structured prompts and timestamps  
- Can use JSON-style prompting  

### Veo 3.1 Fast
- Optimized for speed and stability  
- Best with simple, clear prompts  
- Avoid complex multi-shot logic  

### Kling 3.0 / 2.6
- Strong in realistic human motion  
- Works well with detailed action descriptions  

### Seedance 1.5 Pro
- Strong in stylization and visual consistency  
- Keep motion simple and controlled  

### Hailuo Video 2.3 Pro
- Strong camera control via command syntax  

Example:

```
[Push in]
[Pan left]
[Truck right]
[Zoom in]
[Tracking shot]
[Static shot]
```

Rules:

- Combine: [Pan left, Push in]  
- Sequence: [Push in], then [Pull out]

## 7. Summary

- Structure > randomness  
- Motion realism > visual complexity  
- Camera + lighting = core control  
- Modular prompts enable scaling and automation
