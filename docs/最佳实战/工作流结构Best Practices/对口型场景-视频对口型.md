# Workflow Best Practices: UGC Talking-Head Promo Video (Lip-Sync Structure)

## 1. User Input (Entry Layer)

### Inputs
- `input_1`: influencer_profile (text)
  - desired appearance, vibe, age range, gender, style keywords
- `input_2`: product_image (image)
  - product packshot or clean product reference
- `input_3`: product_website (text)
  - product URL or landing page

### Recommended Input Design
- Start with:
  - one human / influencer definition input
  - one product visual grounding input
  - one product information input
- This combination ensures:
  - influencer consistency
  - product consistency
  - message consistency

---

## 2. Node-Level IO Mapping

### Node A — Product Highlight / Feature Extractor (Text + Image + Text → Text)

**Inputs**
- `input_1`: influencer_profile
- `input_2`: product_image
- `input_3`: product_website

**Output**
- `A_output_1`: product_marketing_brief

**Expected Content**
- product highlights
- core selling points
- positioning angle
- audience-relevant benefits
- possible influencer fit

**Role**
Shared semantic layer. Converts raw inputs into a reusable product brief for both visual generation and script generation.

---

### Node B — UGC Describer (Text + Image + Text → Text)

**Inputs**
- `input_1`: influencer_profile
- `input_2`: product_image
- `A_output_1`: product_marketing_brief

**Output**
- `B_output_1`: ugc_image_prompt

**Expected Content**
- influencer look and feel
- product integration logic
- pose / framing / holding behavior
- UGC-style image description for downstream character generation

**Role**
Transforms the shared marketing brief into a visual prompt for the influencer hero frame.

---

### Node C — Talking Script (Text → Text)

**Inputs**
- `A_output_1`: product_marketing_brief

**Output**
- `C_output_1`: talking_script

**Expected Content**
- short-form speaking script
- hook + message + value point
- spoken language suitable for lip-sync generation

**Role**
Produces the narrative layer of the final promo asset.

---

### Node D — Character / Hero Frame Generation (Image + Text → Image)

**Inputs**
- `input_2`: product_image
- `B_output_1`: ugc_image_prompt

**Output**
- `D_output_1`: influencer_product_image

**Expected Content**
- one influencer-style still image
- product visible in hand or naturally integrated
- stable face and composition suitable for animation

**Role**
Creates the base frame for the later talking-head video.

---

### Node E — Text to Speech (Text → Audio)

**Inputs**
- `C_output_1`: talking_script

**Output**
- `E_output_1`: voice_audio

**Expected Content**
- spoken audio version of the script
- stable pacing for lip-sync compatibility

**Role**
Converts the text script into the driving audio signal for the lip-sync stage.

---

### Node F — Image to Video (Image + Text → Video)

**Inputs**
- `D_output_1`: influencer_product_image
- optional motion instruction embedded in node prompt

**Output**
- `F_output_1`: base_video

**Expected Content**
- short video clip derived from the influencer still
- natural motion
- stable framing
- product still visible

**Role**
Creates the base moving footage used before lip-sync.

---

### Node G — Video to Lip-Sync (Video + Audio → Video)

**Inputs**
- `F_output_1`: base_video
- `E_output_1`: voice_audio

**Output**
- `G_output_1`: final_lip_sync_video

**Expected Content**
- final speaking promo video
- synchronized mouth movement
- product-holding or product-integrated talking-head result

**Role**
Final fusion node. Merges character motion, visual identity, and spoken script into the final ad-ready asset.

---

## 4. Structural Interpretation

### 4.1 Shared Semantic Layer First

`Node A` is the shared semantic dependency.

It should run first because it produces the reusable product brief that powers:
- visual planning (`Node B`)
- copy / script generation (`Node C`)

This ensures that:
- the visual angle
- the spoken message
- the product positioning

all stay aligned.

---

### 4.2 Visual Branch and Audio Branch Are Separate

After `Node A`, the workflow splits into two major branches.

**Branch 1 — Visual Identity Branch**
- `Node A` → `Node B` → `Node D` → `Node F`

Responsible for:
- influencer appearance
- product placement
- visual realism
- base motion footage

**Branch 2 — Spoken Message Branch**
- `Node A` → `Node C` → `Node E`

Responsible for:
- message architecture
- spoken script
- synthesized voice audio

---

### 4.3 Final Merge Happens at Lip-Sync

`Node G` is the final assembly point.

It merges:
- the moving character video (`F_output_1`)
- the generated speaking audio (`E_output_1`)

This is the core difference versus a simpler image-to-video talking-head structure:
- the workflow does not rely on native speech generation inside the video model
- instead, it uses an explicit audio-first lip-sync pipeline

---

### 4.4 Dependency Rules

**Hard dependencies**
- `Node B` cannot run before `Node A`
- `Node C` cannot run before `Node A`
- `Node D` cannot run before `Node B`
- `Node E` cannot run before `Node C`
- `Node F` cannot run before `Node D`
- `Node G` cannot run before both:
  - `Node F`
  - `Node E`

**Why this matters**
This dependency design makes the workflow:
- more controllable
- easier to debug
- easier to rerun partially
- easier to optimize node by node

---

## 5. Why Use This Structure Instead of Direct Talking-Head Generation

This workflow is especially useful when:
- you want stronger control over the spoken voice
- you want to choose or customize TTS separately
- you want more flexibility in audio style
- you want a clearer separation between:
  - character generation
  - motion generation
  - voice generation
  - lip-sync fusion

Compared with a direct image/script → talking-head structure, this design is more modular and easier to tune.

---

## 6. Workflow Design Best Practices

### 6.1 Keep Character Identity Stable Early

The influencer image should be locked in before motion and lip-sync.

That is why `Node D` sits before both:
- `Node F`
- `Node G`

This reduces identity drift in the final video.

---

### 6.2 Treat Audio as a First-Class Asset

Do not treat voice as an afterthought.

Because this workflow uses a dedicated TTS node, the audio can be:
- changed independently
- improved independently
- localized independently
- reused across multiple video variants

---

### 6.3 Use Lip-Sync as the Final Fusion Layer

The lip-sync node should be the final stage, not the first.

This keeps the workflow cleaner:
- image defines identity
- video defines motion
- audio defines message
- lip-sync fuses them into the final asset

---

### 6.4 Optimize Each Branch Independently

Possible iteration points:

**Visual branch**
- regenerate influencer image (`Node D`)
- adjust image prompt (`Node B`)
- regenerate base motion (`Node F`)

**Audio branch**
- rewrite script (`Node C`)
- regenerate voice (`Node E`)
- swap voice style or language

This modularity is a major advantage.

---

## 7. Generalizable Pattern

This structure can be generalized to:
- AI influencer promo videos
- talking avatar ads
- creator-style ecommerce promos
- multilingual talking-head campaigns
- spokesperson videos with reusable voice assets

---

## 8. Abstract Workflow Template

```text
[Input Layer]
(influencer definition + product image + product info)

→ [Shared Insight Layer]
(product brief / positioning)

→ Branch 1: [Visual Planning → Character Image → Base Motion Video]
→ Branch 2: [Script Generation → Voice Audio]

→ [Lip-Sync Fusion Layer]
(final speaking video)
```

---

## 9. Key Takeaways

1. Start with a shared product brief.
2. Split the workflow into visual and audio branches.
3. Generate the influencer image before generating motion.
4. Generate the script before generating voice audio.
5. Use lip-sync as the final merge node.
6. Keep dependencies explicit at every step.
7. Treat this workflow as a modular alternative to direct talking-head generation.
8. This structure is especially strong when voice control and reusability matter.
