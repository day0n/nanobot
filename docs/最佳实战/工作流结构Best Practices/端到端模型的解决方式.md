# Workflow Best Practices: UGC Talking-Head Ad (Explicit IO Mapping)

## 1. User Input (Entry Layer)

### Inputs
- input_1: product_image (image) — web app UI screenshot
- input_2: product_url (text)
- input_3: target_audience (text)

---

## 2. Node-Level IO Mapping (IMPORTANT)

### Node A — Pain Point Analysis (Text → Text)

**Inputs:**
- product_url (input_2)
- target_audience (input_3)

**Output:**
- A_output_1: structured_analysis
  - pain_points
  - value_props
  - emotional_triggers

---

### Node B — UGC Image Prompt Generation (Text → Text)

**Inputs:**
- structured_analysis (A_output_1)
- product_image (input_1)
- target_audience (input_3)

**Output:**
- B_output_1: ugc_image_prompt
  (single hero-frame description)

---

### Node C — Hook & Script Generation (Text → Text)

**Inputs:**
- structured_analysis (A_output_1)
- target_audience (input_3)

**Output:**
- C_output_1: hook
- C_output_2: script

---

### Node D — Character Image Generation (Image → Image)

**Inputs:**
- product_image (input_1)  ← reference grounding
- ugc_image_prompt (B_output_1)

**Output:**
- D_output_1: character_image

---

### Node E — Talking Head Video Generation (Image + Text → Video)

**Inputs:**
- character_image (D_output_1)
- script (C_output_2)

**Output:**
- E_output_1: talking_head_video

---

---

## 4. Key Structural Insights

### 4.1 One Source of Truth

- structured_analysis (Node A output) is the **core shared dependency**
- It feeds BOTH:
  - visual generation (Node B)
  - script generation (Node C)

→ This ensures:
- message consistency
- visual + script alignment

---

### 4.2 Branching Before Execution

Workflow splits into 2 parallel branches:

**Branch 1 (Visual):**
- A → B → D

**Branch 2 (Copy):**
- A → C

Then merges at:
- Node E (video generation)

---

### 4.3 Late Fusion (Critical Design Pattern)

Final node (E) combines:
- visual identity (image)
- narrative (script)

→ This is where “ad” is actually formed

---

### 4.4 Dependency Rules

- Node D MUST NOT run before Node B
- Node E MUST NOT run before:
  - D (image ready)
  - C (script ready)

---

## 5. Generalizable Pattern

This structure is reusable for:

### Any “Character + Narrative” Generation

- UGC ads
- AI influencer videos
- Talking-head explainers
- Avatar-based marketing

---

## 6. Abstract Graph Template

```text
[Input Layer]

→ [Insight Node]
(shared semantic layer)

→ Branch 1: [Visual Planning → Image Generation]
→ Branch 2: [Copy Planning → Script Generation]

→ [Fusion Node]
(video / final asset)
```

---

## 7. Key Takeaways

1. Always build a shared semantic layer first (analysis)
2. Split into visual + narrative branches
3. Keep dependencies explicit
4. Merge only at final generation step
5. Treat image as identity, script as persuasion
