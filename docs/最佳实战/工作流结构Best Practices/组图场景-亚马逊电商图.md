# Workflow Best Practices: Amazon Product Image Set Generation

## 1. User Input (Workflow Entry Point)

The workflow should begin with structured, high-signal inputs:

### Required Inputs
- **Product Reference Image**
  - Clean product image (preferably white background or isolated subject)
- **Product Description (Text)**
  - Title (e.g. Women's Ruched Knot Front Tie Shoulder Sleeveless Dress)
  - Key attributes (material, color, use case, audience)
  - Optional: selling points, brand tone

### Optional Inputs
- Target market (US / EU / etc.)
- Visual style preference (minimal, premium, lifestyle)
- Competitor references

### Best Practice
- Always combine **visual input + textual input**
- Avoid relying on text-only or image-only input
- Treat input as “ground truth” for identity consistency

---

## 2. Generative Workflow Structure

### Core Pipeline

```text
[User Input]
   ↓
(Text-to-Text: Prompt Generation)
   ↓
(Text Splitter: Structured Decomposition)
   ↓
(Image Generation per Prompt)
   ↓
(Result Aggregation / Selection)
```

---

### Step 1: Prompt Generation (Text → Text)

Role:
- “You are a professional Amazon product listing visual designer”

Task:
- Generate a **numbered multi-image set**
- Each item = one image concept

Output format:

```text
Image 01
[description]

Image 02
[description]
```

Best Practices:
- One image per block
- Each block must be self-contained
- Maintain consistent product identity
- Differentiate image function:
  - hero
  - feature highlight
  - use case
  - detail
  - packaging
  - comparison

---

### Step 2: Text Splitting (Text → Structured Units)

Purpose:
- Convert long text into execution-ready units

Instruction:
- Split strictly by numbering
- Remove numbering tokens if needed downstream

Best Practices:
- Each unit must be independently usable
- No cross-references like “same as above”
- Stable format is critical

---

### Step 3: Image Generation (Per Unit Execution)

Input:
- Single prompt per execution

Model:
- Banana Pro / other image model

Enhancements:
- Add rendering instructions:
  - camera (35mm, close-up)
  - lighting (soft daylight)
  - realism (skin texture, fabric detail)
  - composition

Best Practices:
- Keep prompt focused (one scene only)
- Ensure product consistency across all generations
- Avoid mixing multiple scenes in one prompt

---

### Step 4: Result Aggregation

Output:
- Multiple generated images

User Actions:
- Compare results
- Select best-performing visuals
- Download or reuse

Best Practices:
- Enable side-by-side comparison
- Preserve prompt-to-image mapping
- Allow re-generation per single unit

---

## 3. Workflow Design Principles

### 3.1 Linear but Modular

- Each step should be replaceable
- Avoid tightly coupled logic
- Keep nodes atomic:
  - text generation
  - splitting
  - rendering

---

### 3.2 One Node = One Responsibility

Bad:
- One node generates prompts AND splits AND formats

Good:
- Node 1: generate
- Node 2: split
- Node 3: execute

---

### 3.3 Explicit Structure for Automation

- Always enforce numbering
- Always enforce consistent format
- Always enforce one-unit-per-block

This enables:
- Text Splitter compatibility
- Parallel execution
- Retry per unit

---

### 3.4 Human-in-the-loop Friendly

- Allow users to:
  - edit prompts before execution
  - re-run individual images
  - swap models per image

## 5. Abstracted Workflow Template

```text
[Multi-Modal Input]
(image + text)

→ [Text Planning Node]
(generate structured, numbered plan)

→ [Decomposition Node]
(split into atomic units)

→ [Execution Node]
(run generation per unit)

→ [Aggregation Layer]
(compare, select, iterate)
```

---

## 6. Key Takeaways

1. Always start with structured input (image + text)
2. Use text model as planner, not generator only
3. Enforce numbered structure early
4. Split before execution
5. Execute one unit at a time
6. Keep nodes atomic and modular
7. Design for iteration and selection
