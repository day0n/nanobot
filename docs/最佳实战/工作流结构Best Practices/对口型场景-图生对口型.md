# Workflow Best Practices: UGC Talking-Head (Direct Image-to-LipSync Structure)

## 1. User Input (Entry Layer)

### Inputs
- `input_1`: influencer_profile (text)
- `input_2`: product_image (image)
- `input_3`: product_website (text)

---

## 2. Node-Level IO Mapping

### Node A — Product Feature Extraction

**Inputs**
- input_1, input_2, input_3

**Output**
- A_output_1: product_brief

---

### Node B — UGC Visual Description

**Inputs**
- input_1
- input_2
- A_output_1

**Output**
- B_output_1: image_prompt

---

### Node C — Talking Script

**Inputs**
- A_output_1

**Output**
- C_output_1: script

---

### Node D — Image Generation

**Inputs**
- input_2
- B_output_1

**Output**
- D_output_1: influencer_image

---

### Node E — Text to Speech

**Inputs**
- C_output_1

**Output**
- E_output_1: audio

---

### Node F — Image to Lip Sync (Direct Fusion)

**Inputs**
- D_output_1 (image)
- E_output_1 (audio)

**Output**
- F_output_1: final_video

---

## 3. Connection Graph

```text
[input_1, input_2, input_3]
        ↓
   [Node A]
        ↓
  product_brief
     ├────────→ [Node B] → image_prompt → [Node D] → influencer_image ─┐
     └────────→ [Node C] → script → [Node E] → audio ───────────────────┘
                                                                      ↓
                                                         [Node F: Image→LipSync]
                                                                      ↓
                                                             final_video
```

---

## 4. Structural Insights

### 4.1 Single-Step Fusion

- Unlike previous workflow:
  - NO intermediate video generation
- Direct:
  - image + audio → final video

---

### 4.2 Reduced Pipeline Complexity

Previous:
- Image → Video → LipSync

Now:
- Image → LipSync

→ fewer nodes
→ faster execution
→ less cost

---

### 4.3 Dependency Rules

- Node B depends on A
- Node C depends on A
- Node D depends on B
- Node E depends on C
- Node F depends on:
  - D (image)
  - E (audio)

---

## 5. When to Use This Structure

Best for:
- fast ad generation
- high iteration speed
- low-cost production
- simple talking-head scenes

Avoid when:
- complex motion required
- cinematic camera movement needed

---

## 6. Generalizable Pattern

```text
Input
→ Insight Layer
→ Visual + Audio Split
→ Direct Fusion (Image + Audio)
```

---

## 7. Key Takeaways

1. Simplify pipeline when motion is not critical
2. Use image as identity anchor
3. Use audio as message driver
4. Fuse directly when possible
5. Trade flexibility for speed and cost efficiency
