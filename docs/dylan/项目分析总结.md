# nanobot 项目分析总结

**nanobot** 是由香港大学 HKUDS 团队开发的**超轻量级个人 AI 助手框架**，灵感来自 OpenClaw，用 Python 编写，号称用 99% 更少的代码实现了核心 Agent 功能。

---

## 一、核心架构

项目采用**消息总线（Message Bus）驱动**的架构，核心流程：

> 渠道接收消息 → 消息总线路由 → Agent Loop 处理（LLM 调用 + 工具执行循环） → 结果回传渠道

---

## 二、主要功能模块

### 1. Agent 核心引擎（`nanobot/agent/`）

- **Agent Loop**（`loop.py`）：核心处理循环，接收消息 → 构建上下文 → 调用 LLM → 执行工具 → 返回响应，最大迭代 40 轮
- **Context Builder**（`context.py`）：Prompt 构建器，组装系统提示、会话历史、技能等上下文
- **Subagent**（`subagent.py`）：后台子任务执行器，支持异步子任务
- **Skills**（`skills.py`）：技能加载系统，支持内置和自定义技能

### 2. 内置工具集（`nanobot/agent/tools/`）

| 工具 | 功能 |
|------|------|
| `shell.py` | 执行 Shell 命令 |
| `filesystem.py` | 文件读写、编辑、目录列表 |
| `web.py` | Web 搜索 + 网页抓取 |
| `cron.py` | 定时任务管理 |
| `mcp.py` | MCP 协议工具集成 |
| `message.py` | 消息发送 |
| `spawn.py` | 子 Agent 创建 |
| `opencreator.py` | 工作流创建 |

### 3. 多渠道聊天集成（`nanobot/channels/`）

支持 **11 个**聊天平台，全部通过插件化架构实现：

- **Telegram**、**Discord**、**WhatsApp**、**Slack**、**Email**
- **飞书**、**钉钉**、**企业微信**、**QQ**
- **Matrix (Element)**、**Mochat (Claw IM)**

每个渠道继承 `BaseChannel`，实现 `start()`、`stop()`、`send()` 三个核心方法，并支持**第三方插件扩展**（通过 Python entry points）。

### 4. 多 LLM Provider 支持（`nanobot/providers/`）

支持 **20+** LLM 提供商，采用 Provider Registry 架构：

- **云端 API**：OpenRouter、Anthropic、OpenAI、DeepSeek、Gemini、Azure OpenAI、MiniMax 等
- **国内平台**：通义千问（DashScope）、月之暗面（Moonshot）、智谱（Zhipu）、火山引擎、硅基流动等
- **本地模型**：Ollama、vLLM
- **OAuth 登录**：OpenAI Codex、GitHub Copilot

添加新 Provider 只需 2 步：在 `registry.py` 注册 + 在 `schema.py` 加配置字段。

### 5. 会话管理（`nanobot/session/`）

- 基于 **JSONL 文件**存储会话历史
- 支持多会话隔离（按 session key 分文件）
- 无数据库依赖，纯文件系统

### 6. 定时任务与心跳（`nanobot/cron/` + `nanobot/heartbeat/`）

- **Cron 服务**：支持自然语言设置定时任务
- **Heartbeat**：每 30 分钟唤醒，读取 `HEARTBEAT.md` 执行周期任务，结果推送到最近活跃渠道

### 7. MCP 协议支持（`nanobot/agent/tools/mcp.py`）

- 支持 **Stdio** 和 **HTTP** 两种传输模式
- 配置格式兼容 Claude Desktop / Cursor
- 外部 MCP 工具自动发现和注册

### 8. API 服务（`nanobot/api/`）

- **FastAPI** SSE 流式接口：`POST /v1/agent/chat`
- **Clerk JWT 认证**（RS256 公钥验证）
- 健康检查端点 `/health`

### 9. CLI 命令行（`nanobot/cli/`）

| 命令 | 功能 |
|------|------|
| `nanobot onboard` | 初始化配置 |
| `nanobot agent` | CLI 对话模式 |
| `nanobot gateway` | 启动网关（连接所有渠道） |
| `nanobot status` | 查看状态 |
| `nanobot channels login/status` | 渠道管理 |

### 10. 内置技能（`nanobot/skills/`）

- `weather/` — 天气查询
- `summarize/` — 内容总结
- `edit-workflow/` — 工作流编辑
- `workflow-user-guide/` — 使用引导

---

## 三、部署方式

- **本地运行**：`pip install` + `nanobot gateway`
- **Docker / Docker Compose**：提供 `Dockerfile` 和 `docker-compose.yml`
- **Linux 系统服务**：systemd user service
- **多实例**：支持通过 `--config` 参数运行多个独立实例

## 四、技术栈

| 层面 | 技术选型 |
|------|---------|
| 语言 | Python ≥3.11 |
| Web 框架 | FastAPI + Uvicorn |
| LLM 路由 | LiteLLM |
| CLI | Typer |
| 配置 | Pydantic + JSON 文件 |
| 日志 | Loguru |
| 异步 | asyncio |
| WhatsApp Bridge | Node.js + Baileys |
| 存储 | 纯文件系统（JSONL/JSON/Markdown） |

## 五、项目结构

```
nanobot/
├── agent/          # 核心 Agent 逻辑
│   ├── loop.py     #   Agent 循环（LLM ↔ 工具执行）
│   ├── context.py  #   Prompt 构建
│   ├── memory.py   #   持久化记忆
│   ├── skills.py   #   技能加载
│   ├── subagent.py #   后台任务执行
│   └── tools/      #   内置工具（Shell、文件、Web、MCP 等）
├── skills/         # 内置技能（天气、总结、工作流等）
├── channels/       # 聊天渠道集成（支持插件扩展）
├── bus/            # 消息路由总线
├── cron/           # 定时任务
├── heartbeat/      # 主动唤醒
├── providers/      # LLM 提供商（OpenRouter 等 20+）
├── session/        # 会话管理
├── config/         # 配置系统
├── api/            # FastAPI 服务
├── cli/            # 命令行工具
└── prompt/         # Prompt 模板（AGENTS.md, SOUL.md 等）
```

---

## 六、总结

nanobot 是一个**功能完整但代码极简**的个人 AI 助手项目，核心价值：

1. **多渠道统一接入** — 一个 Agent 可同时服务 Telegram、Discord、飞书等多个平台
2. **多 Provider 灵活切换** — 支持几乎所有主流 LLM 提供商
3. **工具丰富** — Shell、文件系统、Web 搜索、MCP、定时任务等
4. **插件化架构** — 渠道和 Provider 都可通过插件扩展
5. **零数据库依赖** — 所有状态基于文件，部署极其简单
